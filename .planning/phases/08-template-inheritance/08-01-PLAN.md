       - SLF4J Logger for edge integrity warnings
       - Public method: `List<PathwayStep> merge(List<PathwayStep> parentSteps, TemplateDiff diff)`
       - Algorithm order per RESEARCH.md Pattern 2:
         a. REMOVALS: filter out parent steps whose stepId is in diff.removals()
         b. OVERRIDES: for each remaining step, if diff.overrides() has a matching stepId, apply field-level override using applyOverride() helper
         c. ADDITIONS: append diff.additions() to the working list
         d. EDGE CHANGES: for each step in the working list, remove prerequisites listed in diff.edgeChanges().remove() (where "to" matches the step's stepId and "from" is in prerequisites), then add prerequisites from diff.edgeChanges().add() (where "to" matches the step's stepId, add "from" to prerequisites)
         e. EDGE VALIDATION: for each step, verify all prerequisites reference a stepId that exists in the final list. Log warning and remove dangling refs.
         f. RENUMBER: reassign stepNumber = index + 1 for all steps in final order

       - Private helper: `PathwayStep applyOverride(PathwayStep step, StepOverride override)`

         - Creates new PathwayStep record with overridden fields. For each of the 12 PathwayStep fields, check if fields.containsKey("fieldName"); if yes, use the override value (with appropriate casting); if no, use the original step's value.
