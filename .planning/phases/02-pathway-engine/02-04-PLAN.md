   - Verify `closeOpenAlerts(TEST_PATIENT_ID)` was called

4. `testPathwayCompletionExitsWorkflow` -- Proves natural completion (D-09):
   - Mock evaluate to return `new PathwayEvaluationResult(true, List.of())` (all complete)
   - Start workflow
   - Wait for workflow result (should complete naturally)
   - Verify evaluate was called
   - Result should be the completed workflow

5. `testQueryMethodReturnsMonitoring` -- Proves query method works while workflow is running:
   - Mock evaluate to return not-complete
   - Start workflow

   - Mock evaluate to return not-complete
   - Start workflow
   - Send 3 signals rapidly
   - Verify evaluate was called at least once (signals coalesce into one wakeup)

IMPORTANT: Time-skipping behavior in TestWorkflowExtension means `Workflow.await(24h, ...)` does NOT actually wait 24 hours. The test environment accelerates time. The brief `Thread.sleep(1000)` calls allow the test environment's internal mechanics to process.

Each test method should use `@Test` annotation from JUnit 5. Use `Mockito.mock()` for activity mocking and `Mockito.verify()` for assertions. Register mocked activities on the worker before `testEnv.start()`.

The test method signatures should accept `TestWorkflowEnvironment testEnv, Worker worker, WorkflowClient workflowClient` parameters injected by the extension.
