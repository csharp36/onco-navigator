**V5__create_physician_overrides.sql** -- Create the physician_overrides table per D-11:

```sql
-- V5__create_physician_overrides.sql
-- Physician override records to suppress false-positive pathway alerts (D-11)
-- When a physician intentionally reorders or skips a pathway step, this record
-- tells the workflow engine not to generate an alert for that patient+step.

CREATE TABLE physician_overrides (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    patient_id UUID NOT NULL REFERENCES patients(id),
    pathway_step_id VARCHAR(100) NOT NULL,
    override_reason TEXT NOT NULL,
    created_by UUID NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

-- Primary lookup: check if override exists for a specific patient + step
CREATE UNIQUE INDEX idx_physician_overrides_patient_step
    ON physician_overrides(patient_id, pathway_step_id);

-- Grant permissions to application user (follows V3 convention)
GRANT ALL ON physician_overrides TO onco_app;
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Flyway migration -> PostgreSQL | Seed data is admin-authored, trusted |
| JSONB templateData -> PathwayStep record | Deserialization boundary -- malformed JSONB could cause runtime errors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | V6 seed migration | mitigate | Pathway templates are version-controlled in Flyway migrations; @Audited on PathwayTemplate entity creates _AUD revision trail for any changes |
| T-02-02 | Information Disclosure | PhysicianOverride entity | accept | Override reason is clinical process text (e.g., "physician ordered alternative sequencing"), not PHI. Patient referenced by UUID only. No encryption needed. |
| T-02-03 | Tampering | PhysicianOverride creation | mitigate | created_by field records the actor UUID; @Audited creates physician_overrides_AUD for change tracking; UNIQUE index on (patient_id, pathway_step_id) prevents duplicate overrides |
| T-02-04 | Denial of Service | JSONB deserialization | accept | Template data is admin-controlled seed data, not user input. Malformed JSONB would fail at application startup (Flyway migration), not at runtime. Low risk. |
</threat_model>
