**StepRow.tsx** -- Shared step row component used by both PathwayDAGView (view mode) and PathwayEditor (edit mode):

```tsx
import { CheckCircle2, Circle, AlertTriangle, MinusCircle, Pencil, Trash2, RotateCcw } from 'lucide-react';
import { Badge } from '@/components/ui/badge';
import { Button } from '@/components/ui/button';
import { Tooltip, TooltipContent, TooltipProvider, TooltipTrigger } from '@/components/ui/tooltip';
import type { PathwayStepStatus } from './types';

interface StepRowProps {
  step: PathwayStepStatus;
  isEditing: boolean;
  onEdit?: () => void;
  onRemove?: () => void;
  onSkip?: () => void;
  onUnskip?: () => void;
}
```

Step icon logic (matching UI-SPEC exactly):
- COMPLETED: `<CheckCircle2 className="h-5 w-5 text-green-600 shrink-0" />`
- ACTIVE + hasActiveAlert: `<AlertTriangle className="h-5 w-5 text-red-500 shrink-0" />`
- ACTIVE (no alert): `<Circle className="h-5 w-5 text-blue-500 shrink-0" />`
- PROPOSED: `<Circle className="h-5 w-5 text-muted-foreground shrink-0 icon-dashed" />` (add `.icon-dashed { stroke-dasharray: 4 2; }` to app.css)
- SKIPPED: `<MinusCircle className="h-5 w-5 text-muted-foreground shrink-0" />`

Status icons get `aria-label` per accessibility contract: "Completed", "Active", "Active - overdue", "Proposed - pending confirmation", "Skipped".

Branching indicator: Receives `depth` and `isLastAtDepth` props. For depth > 0, prefix with monospace connector characters:
- If NOT last at depth: `\u251C\u2500\u2500` (unicode box drawing)
- If last at depth: `\u2514\u2500\u2500`
- Rendered in `<span className="font-mono text-muted-foreground" aria-hidden="true">`

Indentation: `style={{ paddingLeft: step.depth * 24 }}` (24px per depth level per UI-SPEC).

Step name text styling per status (UI-SPEC typography section):
- ACTIVE/COMPLETED: `font-semibold text-sm leading-snug`
- PROPOSED: `text-sm text-muted-foreground` (lighter, no semibold)
- SKIPPED: `text-sm line-through text-muted-foreground`

Below step name: timing info in `text-xs text-muted-foreground mt-0.5`.

SKIPPED steps show `<Badge variant="outline">Skipped</Badge>` with tooltip showing skip reason on hover.

PROPOSED steps show tooltip: "Pending confirmation -- this step was proposed by AI and will not be evaluated until a nurse confirms it."

Row highlight: `bg-amber-50` if `step.hasActiveAlert` (existing pattern).

Minimum touch target: `min-h-[44px]` on the row `<li>`.

Edit mode actions (shown only when `isEditing=true`):
- ACTIVE steps: Edit (Pencil) and Remove (Trash2) icon buttons + "Skip" text button
- COMPLETED steps: Remove (Trash2) only (with warning capability)
- PROPOSED steps: Remove (Trash2) only
- SKIPPED steps: "Restore" (RotateCcw) text button + Remove (Trash2)
- Icon buttons: `aria-label="Edit [Step Name]"` and `aria-label="Remove [Step Name]"`

| Boundary | Description |
|----------|-------------|

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-17 | Tampering | Frontend form submission | mitigate | Zod v4 schema validation on AddStepForm and SkipStepDialog before API call. Server-side @Valid is authoritative. Frontend validation is UX, not security. |
| T-05-18 | Denial of Service | Rapid step add/delete | accept | TanStack Query deduplicates in-flight requests. Server-side @Version prevents concurrent corruption. No rate limiting needed for internal clinical tool (3-5 users). |
| T-05-19 | Information Disclosure | Step data in browser | accept | Step names are clinical process data (not PHI). Patient PHI (name, DOB, MRN) is already displayed on the patient detail page with existing auth controls. No additional exposure from pathway editor. |
</threat_model>
