    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    first_name_encrypted BYTEA NOT NULL,
    last_name_encrypted BYTEA NOT NULL,
    date_of_birth_encrypted BYTEA NOT NULL,

    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    event_date DATE NOT NULL,

    pathway_step_id VARCHAR(100),

    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    created_by UUID NOT NULL

-- Indexes

CREATE INDEX idx_patients_cancer_type ON patients(cancer_type);
CREATE INDEX idx_care_events_patient_id ON care_events(patient_id);

**V3__audit_permissions.sql:**
```sql
-- Restrict audit_log table: application user can only INSERT, never UPDATE or DELETE
-- This makes the audit trail immutable from the application layer

-- Grant INSERT-only to the audit_writer role (created in init-db.sh)
GRANT INSERT ON audit_log TO audit_writer;
GRANT USAGE, SELECT ON SEQUENCE audit_log_id_seq TO audit_writer;

-- Grant SELECT-only to audit_reader role (for compliance queries)
GRANT SELECT ON audit_log TO audit_reader;

-- Assign audit_writer to the application user
GRANT audit_writer TO onco_app;

-- Explicitly revoke dangerous permissions from onco_app on audit_log
REVOKE UPDATE, DELETE, TRUNCATE ON audit_log FROM onco_app;

-- The onco_app user retains full CRUD on all other tables
GRANT ALL ON patients, care_events, alerts, pathway_templates TO onco_app;
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO onco_app;
```

**EncryptionConfig.java** (`com.onconavigator.config`):
```java
@Configuration
public class EncryptionConfig {
    @Value("${onconavigator.encryption.key}")
    private String encryptionKeyBase64;
    
    @Bean
    public SecretKey phiEncryptionKey() {
        byte[] keyBytes = Base64.getDecoder().decode(encryptionKeyBase64);
        return new SecretKeySpec(keyBytes, "AES");
    }
}
```
Add to application-local.yml: `onconavigator.encryption.key: <base64-encoded 256-bit key placeholder>`

**EncryptionConverter.java** (`com.onconavigator.security`):
- Implements `AttributeConverter<String, byte[]>`
- Uses AES-GCM (AES/GCM/NoPadding) with 12-byte random IV prepended to ciphertext
- `convertToDatabaseColumn`: encrypts String to byte[] (IV + ciphertext)
- `convertToEntityAttribute`: extracts IV from first 12 bytes, decrypts remainder
- Injects `SecretKey` via Spring's `ApplicationContextProvider` (static accessor pattern for JPA converters which cannot use constructor injection)
- GCM tag length: 128 bits

|----------|-------------|
| Application to PostgreSQL | App user (onco_app) has restricted permissions on audit_log |

</threat_model>
