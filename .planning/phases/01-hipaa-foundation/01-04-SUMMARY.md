## Authentication Architecture

| Layer | Implementation |
|-------|----------------|
| OIDC client | keycloak-js 26, realm `onco-navigator`, client `onco-navigator-app` |
| PKCE | S256 code challenge — no client secret in browser |
| Token storage | In-memory (keycloak-js default) — not localStorage |
| Token refresh | setInterval(10s) → updateToken(30s buffer) → redirect to login on failure |
| API auth | fetch wrapper reads `keycloak.token`, adds `Authorization: Bearer ${token}` |
| Role source | `keycloak.realmAccess.roles` filtered to `ROLE_*` prefix |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] TypeScript 6 erasableSyntaxOnly incompatibility with class parameter properties**
- **Found during:** Task 1 build (tsc error)
- **Issue:** `class ApiError { constructor(public status: number, ...) }` — TypeScript 6 enables `erasableSyntaxOnly` which disallows type-erased constructor parameter properties
- **Fix:** Refactored ApiError to use explicit field declaration: `status: number; constructor(status, msg) { this.status = status; }`
- **Files modified:** `frontend/src/lib/api-client.ts`
- **Commit:** 7b95def

**2. [Rule 3 - Blocking] TypeScript 6 deprecates `baseUrl` option**
- **Found during:** Task 1 build (tsc error)
- **Issue:** `Option 'baseUrl' is deprecated and will stop functioning in TypeScript 7.0. Specify '"ignoreDeprecations": "6.0"' to silence.`
- **Fix:** Added `"ignoreDeprecations": "6.0"` to tsconfig.app.json compilerOptions
- **Files modified:** `frontend/tsconfig.app.json`
- **Commit:** 7b95def

**3. [Rule 3 - Blocking] shadcn/ui CLI placed components in literal @/ path instead of src/**
- **Found during:** Task 1 shadcn init
- **Issue:** With no prior path alias resolution, `npx shadcn@latest add` created `frontend/@/components/ui/` as a literal directory instead of resolving `@/` to `src/`
- **Fix:** Manually moved all generated component files to `src/components/ui/`, moved hooks to `src/hooks/`, removed the stale `frontend/@/` directory
- **Files modified:** All shadcn UI components
- **Commit:** 7b95def

**4. [Rule 3 - Blocking] routeTree.gen.ts did not exist before first tsc run**
- **Found during:** Task 1 build (Cannot find module './routeTree.gen')
- **Issue:** TanStack Router Vite plugin generates `routeTree.gen.ts` during `vite build`, but `npm run build` runs `tsc -b` first, which fails when the generated file doesn't exist
- **Fix:** Ran `npx vite build` standalone first to generate `routeTree.gen.ts`, then committed it; subsequent `npm run build` passes on first attempt
- **Files modified:** `frontend/src/routeTree.gen.ts` (generated)
- **Commit:** 7b95def
