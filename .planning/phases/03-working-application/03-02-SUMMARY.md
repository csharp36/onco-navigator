## Decisions Made

- Used `staleTime: 0` on `useAlertCount` in addition to `refetchInterval: 30_000` so the alert badge in the sidebar immediately reflects post-mutation state without waiting for the next polling cycle.
- Kept `"use client"` directive in `command.tsx` — it is a string literal in Vite (not Next.js), causes no build error, and is consistent with shadcn's upstream output.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Shadcn CLI installed components to wrong path**
- **Found during:** Task 1 (Install shadcn components)
- **Issue:** `npx shadcn@latest add` wrote component files to `frontend/@/components/ui/` (treating the `@` alias literally) instead of `frontend/src/components/ui/`. The Vite alias `@ -> src` is correctly configured in `vite.config.ts` but shadcn's CLI does not honor it at file creation time.
- **Fix:** Copied all 8 new component files from the erroneous `@/` directory to `src/components/ui/`, then removed the `frontend/@/` directory.
- **Files modified:** All 8 new shadcn component files
- **Verification:** `ls src/components/ui/` showed all components present; `npx vite build` succeeded
- **Committed in:** `9542e69` (Task 1 commit)
