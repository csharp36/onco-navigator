## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API -> Browser cache | Alert data (including decrypted patient names) cached in TanStack Query |
| Browser DOM | PHI displayed in alert cards and dashboard — authenticated users only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-16 | Information Disclosure | PHI in alert cards | accept | Patient name and MRN displayed to authenticated users with NURSE_NAVIGATOR or ADMIN role. This is the intended clinical use case. Data is already decrypted server-side and transmitted over TLS. |
| T-03-17 | Denial of Service | 30-second polling | accept | Three polling hooks (alerts, alert count, dashboard stats) each hit lightweight endpoints. At pilot scale (<10 concurrent users), server load is negligible. |
| T-03-18 | Tampering | Client-side alert filtering | accept | Filter bar operates on already-fetched data — no security impact. Server returns all OPEN alerts; client filters for UX convenience. Authorization enforced server-side. |
</threat_model>
