       return renderer.renderImageWithDPI(0, 300, ImageType.RGB);
   }

**OcrExtractionService.java** (package: `com.onconavigator.service`):

`@Service` class with Logger. Constructor injects `@Value("${onconavigator.tessdata.path:/usr/share/tesseract-ocr/5/tessdata}") String tessDataPath`.

Inner record: `public record OcrResult(String text, int meanConfidence) {}`

Public constant: `public static final int OCR_CONFIDENCE_THRESHOLD = 60;`

1. `public OcrResult performOcr(BufferedImage image)` -- Creates a NEW Tesseract instance per call (thread safety -- Tesseract is NOT thread-safe, virtual threads are enabled):
   ```java
   Tesseract tesseract = new Tesseract();
   tesseract.setDatapath(tessDataPath);
   tesseract.setLanguage("eng");
   tesseract.setOcrEngineMode(1); // LSTM engine
   String text = tesseract.doOCR(image);
   // For confidence, use a simple heuristic: if text length > 100, confidence = 75; else 40
   // Tess4J's getAPI() is not reliably accessible in all builds; use text quality heuristic
   int confidence = (text != null && text.strip().length() > 100) ? 75 : 40;
   return new OcrResult(text != null ? text : "", confidence);
   ```
   Catch `TesseractException`, log warning with non-PHI message, return `new OcrResult("", 0)`.

Note: Do NOT create Tesseract as a Spring bean -- create per-call instances to avoid thread-safety issues with virtual threads (RESEARCH.md Pitfall 6).

Constants: `private static final int MAX_INPUT_TOKENS = 150_000;` and `private static final int CHARS_PER_TOKEN_ESTIMATE = 4;`

1. `@CircuitBreaker(name = "claude-api", fallbackMethod = "classifyFallback")` on:
   `public DocumentClassification classify(String extractedText)`:
   - If `!classificationEnabled`, log info "Document classification disabled (BAA not in place)", return null
   - Truncate text via `truncateToTokenBudget(extractedText)`
   - Call:
     ```java
     return classificationClient.prompt()
         .user(u -> u.text(ClassificationPrompts.USER_TEMPLATE)
                     .param("documentText", truncated))
         .call()
         .entity(DocumentClassification.class);
     ```
   - Catch Exception inside the method body for non-circuit-breaker errors (malformed JSON, etc.), log `"Document classification failed: " + e.getMessage()` (NEVER log extractedText), return null.

2. Fallback method: `public DocumentClassification classifyFallback(String extractedText, Exception e)` -- logs warning `"Claude classification CB open: " + e.getMessage()`, returns null. Must be public for Resilience4j proxy.

3. `private String truncateToTokenBudget(String text)` -- if `text.length() > MAX_INPUT_TOKENS * CHARS_PER_TOKEN_ESTIMATE`, log warning with length (not content), return `text.substring(0, maxChars)`.

**DocumentPatientMatchService.java** (package: `com.onconavigator.service`):

`@Service` class with Logger. Constructor injects `PatientRepository patientRepository`, `HmacTokenService hmacTokenService`.

1. `public MatchResult matchPatient(DocumentClassification classification)` -- Orchestrates multi-field matching per D-05:
   - If `classification.mrn()` is not null and not blank:
     - Compute HMAC token: `hmacTokenService.computeMrnToken(classification.mrn())`
     - Query: `patientRepository.findByMrnHmacToken(hmacToken)`
     - If found, return `MatchResult` with status "EXACT", the patient UUID, empty candidates list
   - If MRN match fails or MRN is null, fall back to name+DOB matching per D-08:
     - Load ALL patients: `patientRepository.findAll()` (acceptable at pilot scale <500)
     - For each patient, compare decrypted `firstName + " " + lastName` case-insensitively against `classification.patientName()`
     - If `classification.dateOfBirth()` is not null, also compare against patient's `dateOfBirth` (after parsing to LocalDate)
     - Score: name match alone = MEDIUM confidence, name + DOB match = HIGH confidence
     - Collect all matches, sort by confidence descending
     - If exactly 1 HIGH match, return "EXACT" with that patient
     - If multiple matches, return "CANDIDATES" with up to 5 ranked PatientCandidate entries
     - If no matches, return "NO_MATCH" with empty candidates
   - Log only UUIDs, never patient names or MRNs

Inner record: `public record MatchResult(String status, UUID matchedPatientId, List<DocumentUploadResponse.PatientCandidate> candidates) {}`

Confidence assignment for candidates:
- Name exact match + DOB match = "HIGH"
- Name partial match (contains) + DOB match = "MEDIUM"
- Name match only (no DOB in document) = "LOW"

Import `DocumentUploadResponse.PatientCandidate` from `com.onconavigator.web.dto`.

**DocumentProcessingService.java** (package: `com.onconavigator.service`):

`@Service` class with Logger and `@Transactional`. Constructor injects:
- `PdfExtractionService pdfExtractionService`
- `OcrExtractionService ocrExtractionService`
- `ClaudeVisionService claudeVisionService`
- `DocumentClassificationService classificationService`
- `DocumentPatientMatchService patientMatchService`
- `ClinicalDocumentRepository documentRepository`

1. `public DocumentUploadResponse processUpload(MultipartFile file, UUID preSelectedPatientId, UUID actorId)`:
   - Validate file: not empty, size <= 20MB, content type is PDF/JPEG/PNG. Throw `ResponseStatusException(BAD_REQUEST)` on failure.
   - Read file bytes: `file.getBytes()`
   - Determine content type from `file.getContentType()`
   - **Text extraction pipeline** (per D-01, D-02, D-03):
     - If content type is `application/pdf`:
       - Try PDFBox: `pdfExtractionService.hasSelectableText(bytes)` ? `pdfExtractionService.extractText(bytes)` : proceed to OCR
       - If PDFBox yields < 50 chars, render first page to image, run `ocrExtractionService.performOcr(image)`
       - If OCR confidence < `OcrExtractionService.OCR_CONFIDENCE_THRESHOLD`, try `claudeVisionService.extractTextFromImage(imageBytes, IMAGE_PNG)`
     - If content type is `image/jpeg` or `image/png`:
       - Convert to BufferedImage, run OCR
       - If OCR confidence < threshold, try Claude vision
     - If all extraction methods yield empty text, set extractedText to null (blank wizard path per D-04)
   - **Classification** (per D-13):
     - If extractedText is not null and not blank, call `classificationService.classify(extractedText)`
     - If classification returns null (circuit breaker open or failure), classification result is null (frontend shows manual dropdown per D-16)
   - **Patient matching** (per D-05, D-06, D-07):
     - If `preSelectedPatientId` is not null (dropped on patient detail page per D-09), skip matching -- use that patient
     - If `preSelectedPatientId` is null and classification is not null, call `patientMatchService.matchPatient(classification)`
     - If classification is null, return NO_MATCH status
   - **Persist ClinicalDocument entity:**
     - Create ClinicalDocument, set all fields (content = file bytes, extractedText = extracted text, documentType, classificationSource, etc.)
     - Set `createdBy = actorId`
     - If preSelectedPatientId is provided, set `patient` via repository lookup
     - Save via `documentRepository.save(doc)`
   - **Return DocumentUploadResponse** with documentId, classificationResult, patientMatchStatus, candidates, matchedPatientId
   - Log: `log.info("Document {} processed: type={} matchStatus={}", doc.getId(), documentType, matchStatus)` -- no PHI in log

For image-to-byte-array conversion (PNG for Claude vision), use:
```java
ByteArrayOutputStream baos = new ByteArrayOutputStream();
javax.imageio.ImageIO.write(image, "png", baos);
byte[] imageBytes = baos.toByteArray();
```

| Threat ID | Category | Component | Disposition | Mitigation Plan |
