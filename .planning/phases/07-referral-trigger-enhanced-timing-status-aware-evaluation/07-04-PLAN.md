The test class should contain these test methods:

**1. `testReferralReceivedAtUsedAsRootAnchor()`**
- Setup: Patient with `referralReceivedAt` set to 10 days ago, `diagnosisDate` set to 30 days ago. One ACTIVE root step (no prerequisites) with `windowDays=14`.
- Expected: Anchor date uses referralReceivedAt (10 days), NOT diagnosisDate (30 days). Since 10 < 14, no MISSING_EVENT alert fires.
- Verify: `alertRepository.save()` never called.

**2. `testDiagnosisDateFallbackWhenNoReferral()`**
- Setup: Patient with `referralReceivedAt` = null, `diagnosisDate` set to 20 days ago. One ACTIVE root step with `windowDays=14`.
- Expected: Falls back to diagnosisDate. 20 > 14 = MISSING_EVENT fires.
- Verify: `alertRepository.save()` called once with AlertType.MISSING_EVENT.

**3. `testScheduledEventSuppressesMissingEvent()`**
- Setup: Patient with diagnosisDate 20 days ago. One ACTIVE root step with `windowDays=14`. One CareEvent with status=SCHEDULED, matching eventType.
- Expected: MISSING_EVENT is suppressed because a SCHEDULED event exists for this step.
- Verify: No alert with type MISSING_EVENT is saved. (DEADLINE_APPROACHING might fire if conditions met â€” that's acceptable.)

**4. `testCancelledEventFiresImmediateAlert()`**
- Setup: One ACTIVE root step. One CareEvent with status=CANCELLED, matching eventType.
- Expected: CANCELLED_EVENT alert fires immediately.
- Verify: `alertRepository.save()` called with AlertType.CANCELLED_EVENT.

**5. `testCancelledMutuallyExclusiveWithDelayed()`** (Pitfall 7)
- Setup: One ACTIVE root step with `windowDays=5`. Patient diagnosisDate 10 days ago (window exceeded). One CareEvent with status=CANCELLED, matching eventType.
- Expected: CANCELLED_EVENT fires but DELAYED_EVENT does NOT fire (mutual exclusion).
- Verify: Only one alert saved, and it is CANCELLED_EVENT (not DELAYED_EVENT).

**6. `testDeadlineApproachingFiresAt48Hours()`**
- Setup: One ACTIVE root step with `windowDays=12`. Patient diagnosisDate 10 days ago (12-10=2 days left). No care events.
- Expected: DEADLINE_APPROACHING alert fires (daysLeft=2, within 0-2 range).
- Verify: `alertRepository.save()` called with AlertType.DEADLINE_APPROACHING.

**7. `testSchedulingUnconfirmedAfter7DaysFromReferral()`** (D-11)
- Setup: Patient with `referralReceivedAt` = 10 days ago. Root step (no prereqs). CareEvent with status=SCHEDULED, `schedulingConfirmed`=false, matching eventType.
- Expected: 10 > 7 = SCHEDULING_UNCONFIRMED fires for root step.
- Verify: Alert saved with AlertType.SCHEDULING_UNCONFIRMED.

**8. `testSchedulingConfirmedSuppressesUnconfirmedAlert()`**
- Setup: Same as test 7 but `schedulingConfirmed`=true on the CareEvent.
- Expected: No SCHEDULING_UNCONFIRMED alert.
- Verify: No alert saved with type SCHEDULING_UNCONFIRMED.

**9. `testResultsNotReadyFires()`** (D-08/D-09)
- Setup: Patient with two care events: (1) SCHEDULED CONSULTATION with eventDate = 5 days from now, (2) SCHEDULED PATHOLOGY_REPORT with expectedCompletionDate = 10 days from now (after the visit). No open RESULTS_NOT_READY alert exists.
- Expected: RESULTS_NOT_READY fires because result completion date (10 days) is after visit date (5 days), within 14-day lookahead.
- Verify: Alert saved with pathwayStepName `"__RESULTS_NOT_READY__"` and AlertType.RESULTS_NOT_READY.

**10. `testResultsNotReadyDoesNotFireWhenResultsBeforeVisit()`**
- Setup: Same as test 9 but PATHOLOGY_REPORT expectedCompletionDate = 3 days from now (before the visit at 5 days).
- Expected: No RESULTS_NOT_READY alert (results will be ready before the visit).
- Verify: No alert with type RESULTS_NOT_READY is saved.
